**工具页面 SEO 规则（默认）**

- **目标**：确保每个工具页面的 meta 描述（tool_*_description）不仅概述工具用途，还必须包含：
  - 工具的计算 / 处理步骤或所用的公式（计算过程），以及
  - 至少一个简短的示例或应用场景（逻辑案例）。

- **理由**：描述中包含计算过程与示例能提高页面对搜索意图的相关性，有助于搜索引擎更好地理解工具的用途与输出，并提升点击率。

- **要求（实现细化）**：
  - `tool_*_description` 字符串长度应足够（建议 >= 80 字符），并显式提及“计算/步骤/公式/示例”类关键词（按语言本地化）。
  - `tool_*_article` 可继续用于更长、更详细的说明与多个示例。

- **验证**：仓库已添加 `scripts/validate-tool-seo.mjs`，它会对 `src/site/i18n/*.ts` 中的 `tool_*_description` 进行启发式检查（多语言关键词匹配）。

- **工作流建议**：在 CI 或构建前运行 `npm run lint:seo`（可选），若校验失败，应由翻译作者补充描述文本以满足规则。
