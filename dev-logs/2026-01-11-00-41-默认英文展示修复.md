摘要：修复默认语言设为英文时仍默认展示中文的问题。

【question】
修改系统默认为英文展示，但目前网站默认为中文

【try to solve】
已修复 Worker 端默认语言判断与 URL 前缀规则：默认语言不再被写死为中文，而是以 `SITE_DEFAULT_LANG`（当前为 `en`）为准；当浏览器偏好中文时会从 `/` 重定向到 `/zh/`，无 `Accept-Language` 或偏好英文时 `/` 直接返回英文页面；同时修复了静态站点构建与客户端脚本中对“中文永远无前缀”的假设，并重新构建站点以保证导航链接与路径前缀一致。

【执行操作】
- 修改：`src/index.ts`
- 修改：`src/pages/websiteHeadersPage.ts`
- 修改：`scripts/site/config.mjs`
- 修改：`scripts/site/layout.mjs`
- 执行：`env SITE_LANGS=zh,en SITE_DEFAULT_LANG=en npm run build:site`
